# ADR Template

Use this template for architecture-significant decisions.

```md
# ADR-0000: Short decision title

Status: Proposed | Accepted | Deprecated | Superseded by ADR-0000 | Rejected
Date: YYYY-MM-DD

## Context
Describe the problem, constraints, forces, and why the decision is needed now.

## Decision
State the decision clearly. Prefer one or two direct paragraphs.

## Consequences
Describe positive, negative, operational, security, testing, and migration consequences.

## Alternatives considered
- Option A: why rejected or deferred.
- Option B: why rejected or deferred.

## Follow-up work
- Implementation tasks.
- Documentation updates.
- Tests or validation gates.
```
